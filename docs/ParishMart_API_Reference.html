Authentication Scheme

All API-to-API communication between ParishMart and the Laravel marketplace uses HMAC-SHA256 request signing. Every request must include four HTTP headers. The same algorithm is used in both directions — our calls to you, and your calls to us.
Required headers on every request:

X-Api-Key — Pre-shared API key identifying the caller.
X-Timestamp — Unix epoch seconds (integer). Requests older than 5 minutes are rejected.
X-Nonce — 32-character hex random string. Prevents replay attacks.
X-Signature — HMAC-SHA256 hex digest of the canonical string (see below).
X-Idempotency-Key — UUID, required on POST requests only. Prevents duplicate creation on retries.
Canonical String Construction
// Canonical string joined with newlines:
canonical = METHOD + "\n" + PATH + "\n" + TIMESTAMP + "\n" + NONCE + "\n" + SHA256(body)

// body = JSON.stringify(requestBody) for POST, or "" for GET
// X-Signature = HMAC-SHA256(canonical, API_SECRET).hex

// Example (GET /api/v1/external-api/parishes):
canonical = "GET\n/api/v1/external-api/parishes\n1743800000\nabc123def456...\ne3b0c44..."

Part A — Our APIs (ParishMart Backend)

These endpoints are hosted on the ParishMart Node.js/Express API (port 3001, AWS). They are the interfaces the Laravel marketplace can call to query registered vendors and stores, and the onboarding entry points for new registrations. All /api/external/* routes require HMAC-SHA256 authentication. Rate limit: 60 req / 15 min per IP.

Part B — APIs We Need from the Laravel Marketplace

The sections below document every endpoint we need the Laravel marketplace to expose so the ParishMart mobile marketplace (Next.js) can replace its current static mock data with live data. For each endpoint we specify the minimum required response fields. Authentication should use the same HMAC-SHA256 scheme described in Section 1.
To the Laravel team: Please provide base URL, versioning strategy, and confirm which fields are available in your existing data model. Fields marked Required are essential for the mobile UI to render correctly. Fields marked Optional are used for richer detail pages and can be added later.
